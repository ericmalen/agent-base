Note: the `base-check` skill deliberately does NOT bundle the audit engine вҖ”
during setup it runs `.claude/agent-base-setup/scripts/audit.mjs`; after
teardown it uses an Agent Base checkout (clone URL in the `.claude/agent-base.json` marker).

Marker fields (R-50): required вҖ” `standard` (semver), `toolRepo`, `setupAt`,
`githubCodeReview`; recommended вҖ” `pin` (release tag), `lastSyncedAt`.
Baseline copies (the permanent skills/agents above) are owned by
`sync-baseline`: upgrades replace them wholesale from the pinned release and
bump `pin`/`lastSyncedAt` вҖ” local edits to baseline files surface as
conflicts, never silently overwritten.

## Compat variant (nested AGENTS.md instead of rules/)

Chosen at setup when the team uses other AGENTS.md-ecosystem tools
(Cursor, Codex, Gemini CLI, вҖҰ). Replaces `.claude/rules/` вҖ” never combined
with it (R-53).

```

    в”Ӯ                              one scope                          R-13, R-14, R-16

## Orchestration layer (CONDITIONAL)

Present ONLY in repos that ran orchestration generation (marker:
`docs/orchestration/generation-manifest.json`); plain setup creates none of
it. Generated by the kit's scaffolder from a validated blueprint via pure slot
substitution вҖ” never hand-authored; every generated file is recorded
(path + sha256) in the manifest, and drift is checked against it
(`drift-checker`, run from an Agent Base clone).

```

в”Ӯ                                  repo-root addition
в”Ӯ
в”ңв”Җв”Җ .claude/
в”Ӯ   в”ңв”Җв”Җ agents/<generated>.md      orchestrator + one agent per
в”Ӯ   в”Ӯ                              blueprint specialist               R-27..R-37
в”Ӯ   в””в”Җв”Җ skills/<generated>/        paired skills per specialist       R-17..R-26
в”Ӯ
в””в”Җв”Җ docs/orchestration/
    в”ңв”Җв”Җ repo-profile.json          discovery output
    в”ңв”Җв”Җ decisions.json             policy answers (source of truth)
    в”ңв”Җв”Җ decisions.md               rendered companion вҖ” never hand-edited
    в”ңв”Җв”Җ blueprint.json             generation input (validated pre-handoff)
    в”ңв”Җв”Җ generation-manifest.json   sha256 record of every generated file

Generated agents and skills land in the standard `.claude/` homes and obey the
same rules as authored ones. The lifecycle skills every setup installs
(`retro`, `log-report`, `eval-runner`) operate on these surfaces; the
discovery/generation meta-assets stay kit-side and are never part of the
target layout.

## Load model (why the layout is shaped this way)

Always-on, every interaction: `AGENTS.md` (Copilot natively; Claude Code via the
CLAUDE.md shim). Per-path, automatic: `.claude/rules/<scope>.md` (both tools) or
nested AGENTS.md (compat). On-demand, model-selected: skills via frontmatter
description scan вҶ’ SKILL.md on activation вҶ’ siblings via Markdown links. Delegated:
agents, loaded only when invoked. Enforced: settings + hooks from
`.claude/settings.json`, read by both tools.

The cost gradient drives every size cap: always-on content is the most expensive
(R-02), per-path next (R-13/R-52), on-demand cheapest (R-20 generous by
comparison). Detailed rationale: `docs/explanation/why-this-way.md`.
